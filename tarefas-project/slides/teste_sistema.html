Teste de Sistema: Simulação do Fluxo de Usuário Final

1
Autenticação (Login)
POST /api/login
Enviar credenciais válidas (email e senha) para obter um token de autenticação.
✓ Status HTTP: 200 OK | Token recebido com sucesso
2
Acesso Não Autorizado (Falha Esperada)
GET /api/tarefas (sem token)
Tentar acessar rota protegida sem enviar o cabeçalho Authorization.
✗ Status HTTP: 401 Unauthorized | Acesso negado conforme esperado
3
Criação de Tarefa (Sucesso)
POST /api/tarefas
Enviar dados de tarefa com cabeçalho Authorization: Bearer <token>.
✓ Status HTTP: 201 Created | Tarefa criada com sucesso
